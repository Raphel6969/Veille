# Design-partner demo program (v0.2.0)

Goal: run 5ā€“10 design-partner demos on the `v0.2.0` release and capture feedback
on five questions (cost-tier trust, intervention safety, cacheable repeats,
freshness rules, rerouting appetite). This folder holds the harness, the
objective system-signal baseline, and the feedback instrument.

## What to run

The harness (`scripts/design_partner_demo.py`) runs the cited-market-research
workflow under opt-in flag presets and emits a per-run **observation packet**
mapped to the five questions.

```powershell
# One demo for a partner (they answer the printed questions)
$env:SUPERVISOR_PLAN=1; $env:SUPERVISOR_OPTIMIZE=1; $env:SUPERVISOR_OPTIMIZE_MODE=active; $env:SUPERVISOR_MEMORY=1
python -m scripts.design_partner_demo --scenario expensive --partner P1

# Regenerate the objective system-signal baseline (no humans needed)
python -m scripts.design_partner_demo --matrix
```

Outputs:
- `runs/<partner>__<scenario>__<preset>.json` ā€” full packet per demo.
- `runs/matrix-analysis.json` ā€” all matrix packets.
- `system-signals.md` ā€” objective signals the supervisor emits today
  (machine + human readable), regenerated by `--matrix`.

## Capture feedback

For each demo, copy `capture-template.md`, paste the observed signals from the
packet, and record the partner's responses. Aggregate per the template's
aggregation section.

## Current objective findings (from `system-signals.md`)

- **Q3 (cacheable):** the `expensive` scenario calls `search_competitors` twice
  with an identical query ā€” an exact duplicate. Under `optimize_active`, it is
  served from cache (`cache_served=1`, `savings=$0.002`). `fetch_source` also
  repeats but is **not** flagged cacheable (not idempotent, inputs differ). This
  is the clearest "repeated, cacheable" signal in the demo workload.
- **Q1 (tiers):** plan tier resolves to `high_quality` across scenarios; total
  cost is small ($0.0138ā€“$0.0238) because the demo uses mock models/tools.
- **Q5 (routing):** routing is advisory and annotates `model.requested` with
  `routing_tier` + reason; it never rewrites execution today.
- **Q4 (freshness):** cache default TTL 300s; memory default TTL none
  (no expiry); drift excludes from retrieval, never auto-deletes.
- **Q2 (interventions):** with enforcement off, outputs are observe-only
  `policy.triggered`; `SUPERVISOR_ENFORCE=true` emits `intervention.applied`
  with `action`/`policy_id`/`reason`/`human_review_required`.

## Next step (conditional)

rules (Q4)**, build **cross-run caching** next (durable `CacheBackend`). Keep
**adaptive rerouting recommendation-only** until enough real outcomes prove it
preserves quality.
